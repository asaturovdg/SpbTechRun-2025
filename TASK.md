## Кейс 3: «Система персонализированных подборок для онлайн-магазина»
Требуется разработать интеллектуальную систему для подбора сопутствующих товаров к основным товарам этапа ремонта «White Box» (предчистовая отделка) с функцией непрерывного обучения на основе действий пользователей.

Необходимо реализовать интеллектуальную систему подборки сопутствующих товаров к основному для ремонта этапа White Box с возможным масштабированием системы на другие этапы ремонта (черновая, чистовая).
Также в процессе реализации необходимо обучать систему по результатам действий пользователя в интерфейсе.
Возможный вариант при получении подборки сопутствующих товаров следующий:
- у пользователя будет блок типа Radiobutton с кнопкой, типы ответа: «подошел», «не подошел» и кнопка «оценить».
- по результату действия пользователя товары, которые попали в правильную выборку, становятся более релевантными.
За основу выбран этап ремонта White Box - предчистовая отделка.
В White Box входят работы такие как:
    - монтаж наливного пола
    - монтаж перегородок ГК и газоблоками
    - выравнивание стен
В таблице разбивка по типам работ, а также список основных и сопутствующих товаров.
В списке "сопутки" есть товары дублеры одной подгруппы.
Это сделано для того, чтобы у покупателя был выбор.

# Рекомендательная система для сопутствующих товаров

## 3. Решение можно разбить на этапы и выполнять последовательно

### Этап 1 - Анализ данных и формулирование гипотез

**Задачи:** 
- Изучение товарного фида в формате YML
- Анализ предлагаемой модели поиска решений для дальнейшей разработки
- Формулировка и проверка гипотез о наиболее значимых свойствах для определения схожести товаров в разных категориях

**Результат:** Краткая аналитическая записка к фиальной презентации

**Формат данных:** YML (Yandex Market Language) - специализированный формат на основе XML

**Пример структуры данных:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<yml_catalog date="2025-10-29T15:48:38+03:00">
  <company>МАКСИДОМ</company>
  <url>https://www.maxidom.ru</url>
  <currencies>
    <currency id="RUR" rate="1"/>
  </currencies>
  <categories>
    <category id="1">Товары для дома</category>
    <category id="10" parentId="1">Средства для мытья посуды</category>
  </categories>
  <offers>
    <offer id="9012">
      <name>Средство для посуды PROSEPT Cooky Apple гель 500мл концентрат</name>
      <url>https://www.maxidom.ru/catalog/sredstva-dlya-mytya-posudy/1001578143/</url>
      <price>890</price>
      <currencyId>RUR</currencyId>
      <categoryId>10</categoryId>
      <param name="Количество">100</param>
      <weight>3.6</weight>
    </offer>
  </offers>
</yml_catalog>
```

### Этап 2 - Разработка и обучение модели

**Задачи:**
- Создание и обучение модели для расчета семантической схожести товаров
- Реализация механизма перерасчета рейтинга на основе пользовательских подтверждений
- Повышение рейтинга схожести между товарами при пользовательском выборе из рекомендаций

### Этап 3 - Разработка функционала выдачи рекомендаций

**API-интерфейсы:**

**GET /recommendations/{product_id}**
- Возвращает 20 самых похожих товаров
- Сортировка по рейтингу схожести

**POST /feedback**
- Принимает ID исходного товара и ID выбранного аналога
- Фиксирует пользовательское подтверждение релевантности

### Этап 4 - Визуализация и интерпретация результатов

**Веб-интерфейс должен обеспечивать:**
- Выбор товара из каталога
- Просмотр списка из 20 рекомендованных товаров
- Отображение основных свойств и рейтинга схожести
- Возможность ручного подтверждения релевантности сопутствующих товаров
- Визуализацию признаков, по которым товары определены как сопутствующие

### Этап 5 - Технические характеристики реализации

**Требования к системе:**
- Работа в закрытом контуре
- Использование локальной модели
- Интеграция с платформой Ollama для запуска LLM локально

**Технологический стек:**
- Ollama для локального запуска больших языковых моделей
- Локальная обработка данных без внешних зависимостей
